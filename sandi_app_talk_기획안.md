## 0. 목적 요약 (Executive Summary)

### 문제 정의

샌디톡(/talk)은 주 58,962회 조회(전체 앱의 17.7%)로 홈 다음으로 가장 많이 방문하는 화면이지만, **진입자 72.3%가 게시글을 클릭하지 않고 이탈**한다. 읽을 콘텐츠가 없어서 스캔 후 떠나는 구조다. NPS 3회차 모두 "콘텐츠 부족"이 비추천 상위 불만으로 반복된다. sandi_recategory 파일은 `건물공지(100%)`와 `분실물·자유게시판·이벤트(94%)`가 전 건물 필수 채널임을 확인해준다.

### 핵심 방향

> **"건물이 곧 커뮤니티"** — 72.3% 빈 탭 이탈을 끊고, 매일 읽을 이유를 만든다.

- **샌디 Talk 전면 삭제** — IA·CDO 방향 확정
- **콘텐츠 공백 해소**: 카테고리 5개로 명확화 (건물공지·행사·분실물·자유·구내식당)
- **분실물 3케이스 정책**: 인포메이션 맡김 / 사무실 발견 / 소지 중 — 케이스별 글쓰기 분기
- **블라인드화 방지**: 대나무숲 삭제, 익명 미지원, 비침습 정책 상시 노출
- **홈 연동**: 건물공지 최근 3개 순환 ticker / 인기글 카테고리 배지 카드

```
[건물톡] 탭  ← 네비게이션 탭 명칭: 샌디톡 → 건물톡 (명칭 차후 확정)
│
├── 전체  ← 기본 활성. 모든 카테고리 통합 피드

├── 행사·이벤트  ← 전체 권한 작성
│
├── 분실물  ← 전체 권한 작성 ★ 신규
│     └── 습득 유형 3케이스 분기 (2-6 상세)
│
├── 자유  ← 이용자 자유 글
│     └── 서브태그: 일상 / 꿀팁 / 질문방
│
└── 구내식당  ← 구내식당 등록 건물에만 탭 노출 (50.4% 건물)

      └── 리뷰  ← 이용자 작성 (파인생활>메뉴>식당 리뷰 통합)
```

**삭제 항목**:
- 샌디 Talk 탭 전체 (수다·대나무숲·질문방 등) — IA 확정·CDO 지시
- 냉난방 안내 — 3.4% 초저적용
- 앱 공통 배너 — 유료화 시 부활

### 2-4. 건물공지 → 홈 ticker 연동 (3개 순환)

| 항목 | 정책 |
|---|---|
| 작성 권한 | ROLE_MANAGER, ROLE_ADMIN |
| 홈 노출 형태 | 최근 공지 최대 3개를 위→아래 방향으로 자동 순환 |
| 순환 방향 | 위에서 아래로 슬라이드 (수직 ticker) |
| 순환 속도 | 3초 간격 자동 전환 (개발팀 협의 가능) |
| 공지 1개일 때 | 순환 없이 고정 노출 |
| 공지 2개일 때 | 2개만 순환 |
| 공지 3개 이상 | 최신 3개만 순환 (4번째 이후는 건물공지 탭에서 확인) |
| 공지 없을 시 | 홈 건물공지 영역 미노출 |
| 공지 삭제 시 | ticker 즉시 갱신 (다음 공지로 자동 대체, 없으면 영역 소멸) |
| 탭 동작 | ticker 어디를 탭해도 해당 공지 상세로 이동 (뒤로가기 → 홈) |
| CRUD | 작성자 본인만 수정·삭제 |

**전체 탭 핀(Pin) vs 홈 ticker 정합**  
홈 ticker: 최근 3개 순환 표시  
전체 탭 핀: 최신 공지 1건만 피드 최상단 고정 (별개 동작)

### 2-6. 분실물 케이스별 글쓰기 정책 ★ 신규

분실물은 발생 상황에 따라 3가지 케이스로 분기된다. 글쓰기 진입 시 **습득 유형을 먼저 선택**하고, 선택에 따라 입력 필드가 달라진다.

#### 습득 유형 선택 (글쓰기 최상단 선택 UI)

| 유형 | 레이블 | 설명 |
|---|---|---|
| A | 인포메이션에 맡겼어요 | 습득한 물건을 인포메이션 데스크에 맡긴 경우 |
| B | 발견한 곳에 있어요 | 사무실·복도 등 건물 내에서 발견, 제자리에 둔 경우 |
| C | 제가 가지고 있어요 | 습득 후 본인이 소지 중인 경우 |

## 7. 수락 기준 (AC)

```
[ 건물톡 기본 구조 ]
□ 네비게이션 탭 명칭 변경 (최종 명칭 확정 후 반영)
□ 샌디 Talk 탭 및 하위 카테고리 전체 미노출
□ 건물톡 탭: 전체 / 건물공지 / 행사·이벤트 / 분실물 / 자유 / 구내식당(조건부) 구성

[ 건물공지 + 홈 ticker 3개 순환 ]
□ 건물공지 게시 시 홈 ticker 즉시 갱신
□ 홈 ticker: 최근 3개 공지 3초 간격 수직 순환
□ 공지 1개 → 고정 노출 / 2개 → 2개 순환 / 3개 이상 → 최신 3개만 순환
□ ticker 탭 시 해당 공지 상세 딥링크 → 뒤로가기 → 홈
□ 공지 삭제 시 ticker 즉시 갱신 (없으면 영역 소멸)
□ 전체 탭 피드 최상단에 최신 공지 1건 핀(📌) 고정

[ 분실물 3케이스 글쓰기 ]
□ 글쓰기 진입 시 습득 유형 선택 UI (A/B/C) 최상단 노출
□ 케이스 A: 물건 이름 + 보관 위치 필드 / 연락처 필드 미노출
□ 케이스 B: 물건 이름 + 발견 위치 + 위치 사진 필수(1장+) / 연락처 필드 미노출
□ 케이스 C: 물건 이름 + 소속 정보 필드 / 연락처 필드 미노출
□ 케이스별 제목·내용 placeholder 정확히 전환
□ 해결됨 상태 배지 전환: 게시글 상세 > [상태 변경] 버튼 (작성자만)
□ 해결됨 전환 시 피드 하단 이동 + 흐림 처리

[ 권한·정책 ]
□ ROLE_USER·ROLE_FIELD: 건물공지·구내식당메뉴 글쓰기 버튼 미노출
□ 글쓰기 하단 서브텍스트 상시 노출 + ">" 링크 탭 시 정책 전문 이동
□ 진입 시 BS 인터럽트 없음

[ 구내식당 조건부 ]
□ 구내식당 미등록 건물 → 탭 미노출
□ 구내식당 등록 → 메뉴 / 리뷰 서브탭 분리 (기본 활성: 메뉴)
□ 메뉴 글쓰기: 댓글·별점 옵션 선택 UI 포함
□ 댓글·별점 옵션에 따라 상세 화면 조건부 렌더링

[ 인기글 연동 ]
□ 임계값 도달 시 홈 인기글 카드에 카테고리 배지 포함 노출
□ 홈 카드 탭 → 게시글 상세 → 뒤로가기 → 홈

[ Empty State ]
□ 카테고리별 문구 및 CTA 정확히 노출
```
